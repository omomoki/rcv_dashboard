# OCI Recovery Service MCP Dashboard デモ台本

想定尺: 4〜5分  
トーン: 最初に共有したYouTube動画のデモ部分に寄せる。  
流れ: MCPの概念を短く説明 → 自然言語プロンプトでRecovery Serviceに質問 → さらに「ダッシュボードも作れます」へ展開。

## 全体構成

1. MCPで何が変わるのかを短く説明
2. 自然言語でRecovery Serviceの情報を聞いてみる
3. いくつか質問を変えて、分析できることを見せる
4. 「この情報をダッシュボードにできますか？」と聞く
5. 作成したダッシュボードを見せる
6. セキュリティと活用例で締める

## 0:00-0:30 オープニング

画面: VS Code / Codex のチャット、またはダッシュボードの冒頭を表示。

ナレーション:

「今回は、OCI Recovery Service の情報を MCP 経由で取得して、自然言語で分析し、最後にダッシュボードとして可視化するデモをお見せします。ポイントは、最初からダッシュボードを作るのではなく、まずAIエージェントに質問しながら、Recovery Serviceの状態を確認していくところです。」

「そのうえで、同じ情報を運用者が見やすい形にまとめるとどうなるか、という流れでダッシュボードまで作成します。」

## 0:30-1:00 MCPの説明

画面: チャット画面、または軽く資料スライド風に話す。

ナレーション:

「MCPは Model Context Protocol の略で、AIエージェントと外部システムをつなぐ標準化されたインターフェースです。今回の場合、AIがOCI APIを直接好きに叩くのではなく、Recovery Service MCPサーバーで公開されたツールを通して、保護DBやバックアップ情報を取得します。」

「つまり、運用者はAPIの細かい呼び方を知らなくても、自然言語で “どのDBが保護されていますか？” や “バックアップ容量が多いDBはどれですか？” と聞けるようになります。」

## 1:00-1:35 プロンプト1: 保護DBとヘルスを聞く

画面: チャットにプロンプトを入力する。

貼るプロンプト例:

```text
大阪リージョンのこのコンパートメントで、Recovery Serviceに保護されているデータベースを一覧化して、現在のヘルス状態を要約して。
```

ナレーション:

「まずはシンプルに、どのデータベースがRecovery Serviceで保護されているかを聞いてみます。ここでは大阪リージョンの検証環境を対象にしています。」

「MCP経由でRecovery Serviceの情報を取得して、保護DBの数、ライフサイクル状態、ヘルス状態をまとめて返してくれます。コンソールを何画面も移動しなくても、まず全体像をつかめるのが便利です。」

## 1:35-2:05 プロンプト2: RPO / REDO / 未保護データ時間を聞く

画面: 2つ目のプロンプトを入力。

貼るプロンプト例:

```text
各保護DBについて、未保護データ時間、REDO転送の有無、バックアップ容量をまとめて。要確認のDBがあれば理由も教えて。
```

ナレーション:

「次に、少し運用寄りの質問をしてみます。たとえば、現在の未保護データ時間、REDO転送の有無、バックアップ容量をまとめて、と聞きます。」

「ここでREDOが有効なDBと無効なDB、RPOが長くなっているDB、バックアップ容量が大きいDBが見えてきます。こういう情報は、手動で集めると地味に時間がかかる部分です。」

## 2:05-2:35 プロンプト3: サービス別に分ける

画面: 3つ目のプロンプトを入力。

貼るプロンプト例:

```text
保護DBを ExaDB-XS、ExaDB-D、BaseDB などのサービス種別で分類して、それぞれ何台あるか、REDO有効が何台あるかを教えて。
```

ナレーション:

「さらに、サービス別にも分けてみます。ExaDB-XS、ExaDB-D、BaseDBのように分類して、それぞれ何台保護DBがあるか、REDO有効が何台あるかを聞きます。」

「今回の環境では、ExaDB-XS、ExaDB-D、BaseDB、そしてDB本体の詳細が取れずサービス種別を確定できない不明のDBもあります。このように、不明なものも含めて見える化できるのが重要です。」

## 2:35-3:05 プロンプト4: 容量とランキングを聞く

画面: 4つ目のプロンプトを入力。

貼るプロンプト例:

```text
バックアップ容量が大きい保護DBをランキングで表示して。サービス種別、REDO状態、バックアップ容量も一緒に見せて。
```

ナレーション:

「次に、バックアップ容量が大きいDBをランキングで聞いてみます。容量が大きいDBは、コストや復旧時の影響も大きくなりやすいので、運用上は優先して見たい情報です。」

「このように、質問を少し変えるだけで、同じRecovery Serviceのデータから別の観点の分析ができます。」

## 3:05-3:30 プロンプト5: ダッシュボード化を依頼する

画面: チャットに「ダッシュボード作れる？」というプロンプトを入力。

貼るプロンプト例:

```text
今取得したRecovery Serviceの情報を使って、運用者が見やすいダッシュボードを作って。サービス別サマリ、REDO状態、バックアップ容量ランキング、データベース一覧を含めて。
```

ナレーション:

「ここまで自然言語でいくつか質問してきましたが、毎回チャットで確認するだけでなく、同じ情報をダッシュボード化することもできます。」

「そこで、今取得した情報を使って、サービス別サマリ、REDO状態、バックアップ容量ランキング、データベース一覧を含むダッシュボードを作って、と依頼します。」

## 3:30-4:20 ダッシュボードデモ

画面: 作成済みの `index.html` ダッシュボードを表示。

ナレーション:

「こちらが作成したダッシュボードです。上段では保護対象DB数、バックアップ数、総バックアップ容量、REDO転送、月額コスト見込みを確認できます。」

画面: サービス別サマリへ移動。

「サービス別サマリでは、ExaDB-XS、ExaDB-D、BaseDB、不明に分けて、保護DB数、稼働中、REDO有効、要確認の数を表示しています。リアルタイムREDOのバーは、有効、無効、不明の割合が分かるようにしています。」

画面: サービス別 保護DBリストへ移動。

「サービス別 保護DBリストでは、各サービス配下の保護DBを確認できます。右側のアイコンはREDO状態だけを表示しています。緑が有効、黄色が無効、グレーが不明です。」

画面: 保護データ量が多いDB、バックアップ間隔ウォッチを表示。

「さらに、バックアップ間隔ウォッチや保護データ量が多いDBのランキングも確認できます。RTOは今回のAPIから直接取得できないため、ここでは運用上見たい情報として、容量の大きいDBランキングに置き換えています。」

## 4:20-4:50 データベース一覧と詳細確認

画面: データベース一覧を表示。

ナレーション:

「最後に、データベース一覧です。ここではサービス別にグループ化し、状態、ヘルス、RPO、バックアップ容量、バックアップ回数、REDO状態を確認できます。」

「サマリで気になったサービスやDBがあれば、ここで詳細を確認するという流れです。」

## 4:50-5:15 セキュリティとまとめ

画面: 冒頭のサマリに戻る。

ナレーション:

「重要なのは、AIがOCIを自由に操作しているわけではないという点です。MCPサーバーはOCIの認証と権限に従って動作し、許可された情報だけを取得します。」

「このように、MCPを使うことで、Recovery Serviceの情報を自然言語で確認し、そのままダッシュボードやレポートに展開できます。運用レビュー、日次チェック、監査準備、トラブルシューティングの入り口として活用できると思います。」

## 撮影メモ

- 最初はチャット画面を見せる。いきなり完成ダッシュボードから入らない。
- プロンプトを貼るたびに、回答を長く読まず「こういう観点で返ってきます」と要約する。
- ダッシュボードは「チャットで見えた情報を固定化したもの」として紹介する。
- 元動画のように、途中で「これをPDFにしたり、UIにしたりできます」という一言を入れると自然。
- 5分に収める場合、MCP説明は短めにして、プロンプトデモとダッシュボードに時間を使う。
